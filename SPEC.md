# ワンタップタイマー（仮）実装設計書

> デザイン（色・レイアウト・余白等のビジュアル）は本書のスコープ外。実装者の裁量とする。
> 本書は機能仕様・画面の情報構造・データモデル・技術構成を定義する。

---

## 1. プロダクト概要

- **コンセプト**: ウィジェットからワンタップでタイマーを起動できるiOSタイマーアプリ
- **タイマー種別**: 相対タイマー（「○分後に鳴る」）専用。絶対時刻アラーム（「7:00に鳴る」）・曜日繰り返し・世界時計は**実装しない**
- **主なユースケース**: カップ麺、仮眠、二度寝、勉強、料理、長時間の締切カウントダウン（72時間等）
- **マネタイズ**: 買い切りPro。無料でアプリとして完結し、ヘビーユーザーが課金する設計

### 対応環境

- **iOS 26以上専用**（AlarmKit必須のため）。それ未満へのフォールバックは実装しない
- iPhoneのみ。iPad対応は不要
- 日本語UI（i18n基盤は入れてよいが、v1は日本語のみ）
- ダークモード対応

### 技術スタック

- React Native + Expo（CNG / prebuild前提）
- DB: Drizzle ORM + expo-sqlite
- 状態管理: Zustand
- ネイティブ部分（Swift必須）:
  - AlarmKit連携モジュール
  - WidgetKit拡張（インタラクティブウィジェット、App Intent）
  - ActivityKit（Live Activity）
  - 既存パッケージ（`react-native-nitro-ios-alarm-kit` / `expo-alarm-kit` 等）を評価し、要件を満たすなら採用。満たさなければ `expo-apple-targets` + 自前Swiftで実装
- 課金: StoreKit 2（買い切り非消耗型）。RevenueCat等の採用は実装者判断
- ビルド/配布: GitHub Actions + fastlane（手順は既存docに従う。本書では扱わない）

## 2. ドメインモデル

### 2.1 プリセット（Preset）

登録済みタイマー。タップで即起動する。

| 属性 | 内容 |
|---|---|
| id | UUID |
| icon | アイコン識別子（SF Symbols名 or 絵文字。実装者判断で統一） |
| color | カラー識別子（定義済みパレットから選択） |
| durationSec | 秒数。上限は7日（604800秒）程度を目安 |
| inWidget | ウィジェット表示対象か（bool） |
| sortOrder | 並び順（エリア内での順序） |

- **名前（ラベルテキスト）は持たない。** 識別はアイコン＋色のみ
- `inWidget = true` のプリセットがウィジェットに表示される
- **無料はinWidget枠が3つまで。Proで無制限**（プリセット総数の制限ではなく、ウィジェット表示枠の制限）

### 2.2 実行中タイマー（RunningTimer）

| 属性 | 内容 |
|---|---|
| id | UUID（AlarmKitのアラームIDと対応付ける） |
| presetId | 起動元プリセット（使い捨て起動ならnull） |
| icon / color / durationSec | 起動時点のスナップショット |
| endAt | 終了予定時刻 |
| state | running / paused / finished |
| pausedRemainingSec | 一時停止時の残り秒数（paused時のみ） |

- 複数同時実行を許可する。同一プリセットの多重起動も許可（カップ麺2個など）
- 状態はSQLiteに永続化し、アプリ再起動後も復元する。ウィジェット/Live Activityと共有が必要な最小限の状態はApp Groups経由で共有する

### 2.3 起動履歴（分析用）

| 属性 | 内容 |
|---|---|
| id | 自動採番 |
| presetId | null可（使い捨て起動） |
| durationSec | 起動した長さ |
| startedAt | 起動日時 |
| source | app / widget / liveactivity 等の起動元 |

- タイマー起動のたびに1行記録。完了/キャンセルの別は記録してもよい（任意）

## 3. 機能仕様

### 3.1 タイマー起動

- **プリセット起動**: メイン画面のプリセットをタップ → 確認なしで即起動。誤タップ対策として起動フィードバック（ハプティクス＋視覚）を出す。確認ダイアログは出さない
- **使い捨て起動（今すぐタイマー）**: FAB → 時間設定UI（ダイヤル式）→ スタート。プリセットには保存されない
- 起動時の処理:
  1. AlarmKitにカウントダウンをスケジュール
  2. Live Activityを開始
  3. RunningTimerをDBに記録
  4. 起動履歴を記録
  5. ウィジェットのタイムラインをリロード

### 3.2 終了時の通知（AlarmKit）

- AlarmKitのカウントダウンを使用。これにより:
  - 消音モード・Focus中でも鳴る
  - ロック画面での全画面アラート（停止/スヌーズUI）
  - アプリがkillされていても発火
- 権限: 初回起動オンボーディングでAlarmKit権限をリクエスト。`NSAlarmKitUsageDescription` を設定
- 拒否された場合: タイマー起動時に「アラームが鳴らせない」旨を表示し、設定アプリへの導線を出す
- アラート音: v1はシステム/バンドル内サウンドから1〜数種。カスタム音源インポートはスコープ外
- スヌーズ: AlarmKit標準のスヌーズ挙動に従う（独自実装しない）

### 3.3 一時停止・再開・キャンセル

- **一時停止**: AlarmKitにネイティブの一時停止APIがあるか実機検証する。無い場合は「AlarmKitのアラームをキャンセル → 残り秒数を保持 → 再開時に残り秒数で再スケジュール」で実現する
- 一時停止中はLive Activityの表示も停止状態に更新する
- **キャンセル**: AlarmKitのアラームをキャンセルし、Live Activityを終了、RunningTimerを削除
- ウィジェット・Live Activity・アプリ内のどこから操作しても状態が一致すること（App Groups + タイムラインリロードで同期）

### 3.4 ウィジェット（ホーム画面、iOS 17+インタラクティブ）

- **待機状態**: `inWidget = true` のプリセットをボタンとして表示（アイコン＋色＋時間）。タップ → App Intent経由でアプリを開かずに起動
- **実行中状態**: 表示が切り替わり、残り時間（`Text(timerInterval:)` でOS側がカウントダウン描画）・終了時刻・キャンセルボタンを表示
- サイズ: small / medium の2種。サイズにより表示プリセット数が変わる（具体数は実装時にレイアウトで決定）
- 複数タイマー実行中の表示優先順位: 終了が近い順
- ウィジェット未設置・プリセット0件時のプレースホルダ表示を用意

### 3.5 Live Activity（ロック画面・Dynamic Island）

- タイマー実行中はLive Activityを表示: アイコン・色・残り時間・終了時刻
- Dynamic Island対応(compact / expanded)
- 操作: 一時停止・キャンセル（App Intent）。3.3の検証結果次第で一時停止は省略可
- 複数実行中の扱いはOSの制約に従う（表示できる数に上限がある前提で設計）

### 3.6 プリセット管理

- 作成/編集項目: **時間（ダイヤル式UI）・アイコン・色** の3つのみ。名前入力は無い
- アイコン: 定義済みセットから選択（料理・睡眠・勉強・運動・家事など日常ユースを網羅した20〜40種程度）
- 色: 定義済みパレット8色程度から選択
- 削除可能。実行中タイマーがあるプリセットを削除しても実行中タイマーは継続する（スナップショット方式のため）

### 3.7 メイン画面のエリア分割とドラッグ

- プリセット一覧は2エリアに分かれる:
  - **上エリア**: ウィジェットに出さないプリセット（inWidget = false）
  - **下エリア**: ウィジェットに出すプリセット（inWidget = true）
- 長押しドラッグでエリア間を移動でき、移動 = inWidgetのトグル
- エリア内の並べ替えもドラッグで可能（sortOrder更新）。下エリアの並び順 = ウィジェットでの表示順
- **無料制限**: 下エリアは3枠まで。4つ目をドラッグしようとしたらPro購入導線を表示

### 3.8 Pro（買い切り・非消耗型）

| 機能 | 無料 | Pro |
|---|---|---|
| プリセット作成 | 無制限 | 無制限 |
| ウィジェット表示枠（下エリア） | 3つまで | 無制限 |
| 分析画面 | ロック（プレビュー表示） | 解放 |
| その他全機能 | 利用可 | 利用可 |

- 復元（restore purchases）を設定画面に必ず置く
- 価格は本書では定めない（低単価買い切りを想定）

### 3.9 分析画面（Pro）

- 起動履歴を集計して表示:
  - プリセット別の起動回数（期間: 週/月/累計の切り替え程度）
  - 合計タイマー時間
- 凝った可視化は不要。v1は回数と時間が分かれば良い
- 無料ユーザーには内容をぼかしたロック状態＋Pro導線を表示

### 3.10 設定画面

- アラート音の選択
- Pro購入・復元
- アプリ情報（バージョン、問い合わせ、プライバシーポリシー、利用規約）

## 4. 画面構成（情報構造）

```
メイン画面
├─ ヘッダー: アプリ名 / 分析アイコン / 設定アイコン
├─ プリセット上エリア（ウィジェット外）
├─ プリセット下エリア（ウィジェット表示・3枠制限表示）
├─ FAB（今すぐタイマー）
└─ 実行中ドック（下部）
   └─ 実行中タイマー行 × n（アイコン・残り時間・終了時刻・一時停止・キャンセル）
      ※ ゼロ件時は空状態表示

シート/モーダル
├─ プリセット作成・編集（ダイヤル・アイコン・色・ウィジェット表示トグル・削除）
├─ 今すぐタイマー（ダイヤル・スタートのみ。作成/編集と同一のダイヤル部品を使う）
├─ 分析（Pro）
├─ 設定
└─ Pro購入（ペイウォール）

アプリ外
├─ ウィジェット small / medium（待機 ⇄ 実行中）
├─ Live Activity（ロック画面 / Dynamic Island）
└─ AlarmKit全画面アラート（OS標準）
```

- タブバーは使わない。メイン1画面＋シートで構成する
- ダイヤルUIは「プリセット作成/編集」と「今すぐタイマー」で同一コンポーネントを共有する

## 5. オンボーディング（初回起動）

1. コンセプト説明（1〜2枚。ウィジェットからワンタップ起動）
2. AlarmKit権限リクエスト（「確実に鳴らすため」の文脈で）
3. ウィジェット追加の案内（追加手順の図解または案内。スキップ可）
4. デフォルトプリセットを3つ投入した状態で開始（例: 3分/20分/25分。inWidget = true）

## 6. エッジケース・挙動定義

- **アプリkill中の終了**: AlarmKitが発火するためアラートは鳴る。次回起動時にRunningTimerの状態をendAtと現在時刻から再計算して整合させる
- **時刻変更・タイムゾーン**: endAtはエポック秒で保持し、表示時にローカル変換
- **72時間級の長時間タイマー**: 実行中ドック・Live Activity・ウィジェットで「○日 ○○:○○:○○」形式の表示に対応する
- **ウィジェット枠の超過状態**: Proを購入後に返金等でPro判定が外れた場合、inWidgetが4つ以上残っていても既存分は維持し、新規追加のみブロック
- **AlarmKit権限が後から取り消された場合**: 起動時に権限を確認し、欠けていればバナーで再許可導線を出す

## 7. 要実機検証項目（実装前/実装初期に確認）

優先度順。結果次第で仕様を確定する。

1. **AlarmKitカウントダウンの一時停止**: ネイティブAPIの有無。無ければ「キャンセル→再スケジュール」方式で実装（3.3）
2. **既存パッケージの評価**: `react-native-nitro-ios-alarm-kit` / `expo-alarm-kit` が、カウントダウン・キャンセル・複数同時・Live Activity連携の要件を満たすか。満たさない場合は自前Swiftモジュール
3. **インタラクティブウィジェットのボタン応答ラグ**: App Intent実行〜ウィジェット再描画の体感。許容できる演出（即時の楽観的UI更新等）を検討
4. **AlarmKitの同時スケジュール数上限**: 複数タイマー同時実行の上限有無
5. **Live Activityの複数同時表示**: 表示数の実際の挙動
6. **シミュレータ不可・実機必須の範囲**: AlarmKitは実機必須前提でテスト計画を組む（EAS Build → TestFlight ループ）

## 8. v1スコープ外（実装しない）

- 絶対時刻アラーム・曜日繰り返し・世界時計・ストップウォッチ（カウントアップ）
- iOS 26未満のフォールバック（ローカル通知版）
- スワイプジェスチャーでの今すぐタイマー呼び出し（v1.1候補。FABのみ）
- カスタム音源のインポート
- iPad / Apple Watch対応
- iCloud同期
- 多言語対応（基盤のみ可）

## 9. ストア関連メモ

- アプリ名は仮。「ワンタップタイマー」系の説明的な名前を想定（ASO観点）。最終決定は別途
- App Storeの対応OS表記がiOS 26+になる点を説明文で明示
- プライバシー: 全データ端末内（SQLite）。外部送信なし（課金除く）。Privacy Nutrition Labelは「データ収集なし」で申告できる構成を維持すること（分析SDK等を入れない）
