# Marketing

今すぐタイマーのストア訴求、スクリーンショット方針、SNS投稿、動画台本、価格/アイコンAB案を蓄積する場所。現在の作業状態や引き継ぎは `docs/HANDOFF.md` に置く。

## 目的

- ゴールはApp Storeで公開し、継続的に改善して売上を作ること。
- 実装相談でも、初見の伝わり方、ストアCV、レビュー、課金導線、継続利用への影響を同時に見る。
- ここには仮説・施策案・生成物方針を残し、現在の作業状態は `docs/HANDOFF.md` に残す。

## 基本方針

- 日本向けではアプリ名「今すぐタイマー」を主役にする。
- 英語名 Tappri は必要な場合だけ補助的に使う。
- 訴求軸は「ウィジェットからワンタップでタイマー開始」「よく使う時間をすぐ起動」「消音/集中モードでも鳴る安心感」。
- 絵文字や過剰な装飾は避け、iPhoneユーザー向けに清潔で実用的な見た目にする。
- 「タイマーを開く手間がない」「料理・仮眠・勉強など何度も使う時間をすぐ始められる」を短く伝える。
- AlarmKitの強みは安心材料として使うが、過度に専門用語化しない。

## ストアスクリーンショット

- 素材: `assets/screenshot/*.PNG`
- 出力先: `output/store/`
- 基本サイズ: 1290x2796 PNG
- 生成: Codex Desktop の `$imagegen`（`gpt-image-2`）を使う。
- 日本向け1枚目の主訴求: 「ウィジェットからワンタップでタイマー開始」
- 実機スクショのUIは自然に見せ、内容を過度に改変しない。

## リリース初動方針

- 現在はApp Store審査中。スクリーンショット・ASO・ストア掲載情報は提出済み。
- リリースは自動ではなく手動。審査通過後すぐ公開せず、TikTok/X投稿とタイミングを合わせて初速を作る。
- TikTokはフォロワー約1200人。これまで作ってきたアプリの作り方や個人開発の過程を簡単に解説するアカウント。
- Xはフォロワー約1500人。個人開発者としての人格で発信している。
- TikTokでは「個人開発でこういうアプリを作った」文脈、Xでは個人開発者としての公開報告・開発背景・使いどころを出す。
- まず準備すべきものは、公開日用のTikTok台本、X投稿スレッド、App Storeリンク導線、リリース後24時間の追加投稿。
